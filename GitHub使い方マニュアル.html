🐙 GitHub Pages ガイド

GitHubを使って
審判シフトを公開する

GitHubはHTMLファイルを無料でWeb上に公開できるサービスです。一度アップロードすれば永久に止まらないURLが使えます。LINEで送るだけでスマホから誰でも見られます。

📁 HTMLファイルを用意
→
📤 GitHubにアップロード
→
🔗 URLが発行される
→
📱 LINEで共有
📤
ファイルをアップロードする
新しいファイルを初めて追加する場合
1

GitHubのリポジトリページを開く

ブラウザで以下のURLにアクセスする

https://github.com/nrkyagi1212-coder/soccer-sift
2

「+」ボタンをクリック → 「Upload files」を選ぶ

ファイル一覧の上にある「+」ボタンをクリックする

github.com/nrkyagi1212-coder/soccer-sift
soccer-sift Code ▼ +
Create new file
📤 Upload files ← ここ
3

ファイルをドラッグ＆ドロップ

パソコンのエクスプローラーからHTMLファイルを画面中央の枠にドラッグする。または「choose your files」をクリックしてファイルを選ぶ。

💡
複数ファイルを一度にドラッグしてまとめてアップロードできます。
4

「Commit changes」ボタンを押す

ページ下部に緑のボタンが出るのでクリックする。これでアップロード完了。

Commit changes
5

2〜3分待ってからURLにアクセス

GitHubがページを公開するまで少し時間がかかる。

https://nrkyagi1212-coder.github.io/soccer-sift/アップロードしたファイル名.html
🔄
ファイルを更新する（上書き）
修正したHTMLを再アップロードする場合
⚠️
注意：GitHubは同じファイル名でアップすると自動で上書きされない
同じ名前でアップしても別ファイルとして追加される。確実に更新するには以下の方法を使う。

方法A：既存ファイルを編集して上書き（おすすめ）

1

リポジトリのファイル一覧から対象ファイルをクリック

github.com/nrkyagi1212-coder/soccer-sift
📄soccer_shift_6.html1 hour ago
📄handball_shift_2.html ← クリック
2

右上の鉛筆アイコン（✏️）をクリック

ファイルの中身が表示されたら、右上に鉛筆マークのボタンがある。

3

全選択（Ctrl+A）→ 削除 → 新しい内容を貼り付け

テキストエリアで Ctrl+A で全選択、Delete で削除、新しいHTMLの内容をCtrl+Vで貼り付ける。

4

「Commit changes」を押す

これで同じURLのまま内容が更新される。

方法B：新しいファイル名でアップする（簡単）

修正したHTMLを soccer_shift_7.html のように番号を上げてアップするだけ。URLが変わるのでLINEで再共有が必要。

📝 ファイル名の付け方ルール
競技名_shift_バージョン番号.html
例: soccer_shift_7.html　basketball_shift_4.html
🔗
URLの確認方法
公開URLはファイル名から自動で決まる
📝 URLのパターン
https://nrkyagi1212-coder.github.io/soccer-sift/ファイル名.html
GitHubのファイル名公開URL
soccer_shift_6.html...soccer-sift/soccer_shift_6.html
basketball_shift_3.html...soccer-sift/basketball_shift_3.html
total_ranking_3.html...soccer-sift/total_ranking_3.html
💡
GitHub Pagesのデプロイ状況の確認方法
Settings → Pages → 「Your site is live at ...」と表示されていればOK。
「github-pages」の横に緑の✅が出るまで少し待つ（1〜3分）。
🆕
新しいファイルを追加する
来年度の新競技・新メンバーに対応するとき
1

既存のHTMLをパソコンにダウンロード

GitHubでファイルをクリック → 右上の「⬇️ Download raw file」ボタン

2

テキストエディタで開いて内容を変更

「審判シフトシステム カスタマイズマニュアル」を参照しながら体実メンバー・グループ・時間割を書き換える

3

新しいファイル名で保存

例: soccer_shift_2027.html のようにわかりやすい名前をつける

4

GitHubにアップロード（上記「アップロード手順」参照）

2〜3分後にURLにアクセスして動作確認する

🗑
古いファイルを削除する
不要なファイルが増えてきたら整理する
1

削除したいファイルをクリック

2

右上の「…」ボタン → 「Delete file」をクリック

3

「Commit changes」を押す

URLにアクセスすると404エラーになれば削除成功

⚠️
削除したURLをLINEで共有しているなら注意
そのURLを使っている人がいると404エラーになる。削除前に全員に新URLを周知すること。
⚠️
よくあるトラブル
症状原因対処法
URLにアクセスすると404エラー ①ファイル名のスペルミス
②デプロイがまだ終わっていない		 GitHubのファイル一覧でファイル名を確認。2〜3分待ってから再アクセス
アップロードしたのに古い内容が表示される ブラウザのキャッシュ Ctrl+Shift+R（強制リロード）を押す
「このサイトは停止されました」と表示される Netlifyを使っていた頃の名残。GitHubに移行済みなら関係ない GitHubのURLを使う
Settings → Pagesが見つからない リポジトリの設定画面のメニューの場所 Settings → 左メニューを下にスクロール → 「Code and automation」の中の「Pages」
ファイルをアップしたら番号が増えてしまう（soccer_shift_5.html など） 同名ファイルとして上書きされずに追加される仕様 方法A（編集して上書き）を使うか、新URLをLINEで再共有する
