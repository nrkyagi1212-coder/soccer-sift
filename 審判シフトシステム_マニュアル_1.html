審判シフトシステム
カスタマイズマニュアル

このシステムは来年度以降も使い回せるように設計されています。
HTMLファイルをメモ帳などのテキストエディタで開いて、決まった7箇所を書き換えるだけで別の競技・別の体実メンバーに対応できます。

STEP 1
HTMLファイルをコピー
STEP 2
7箇所を書き換え
STEP 3
GitHubにアップロード
STEP 4
URLをLINEで共有
🛠
必要なもの
1

テキストエディタ

メモ帳（Windows）/ テキストエディット（Mac）でOK。VSCodeがあればなお良い。

2

GitHubアカウント

ファイルをアップロードしてURLを発行するために使用。無料。

3

Firebaseアカウント

リアルタイム同期に使用。既存の taizitu-sift プロジェクトを使い回せる。

4

既存のHTMLファイル（コピー元）

soccer_shift.html などをベースにコピーして改造する。

1
体実メンバーの変更
HTMLファイルの上部にある ALL_STAFF を書き換える
場所 HTMLファイルの <script> 内、上から数行目
変更前（サッカーの例）
var ALL_STAFF=[
  {cls:'5C',name:'大友翔'},
  {cls:'5E',name:'横瀬智哉'},
  {cls:'5D',name:'角川健太郎'},
  {cls:'5B',name:'平田詩'},
  {cls:'4E',name:'矢木誠人'},
  {cls:'4G',name:'藤代真平'},
  ...
];
📝 変更ルール
・cls → そのメンバーのクラス（例: '5C' '4G'）
・name → 名前（姓のみでOK）
・メンバーが増減しても { }, の行を追加・削除するだけ
・最後の行だけカンマ不要（} で終わる）
⚠️
5年生優先機能を使う場合
競技によっては STAFF_5（5年）と STAFF_4（4年）に分けて書く。ドッヂビー・ハンドボールなどがこの形式。その場合は grade:5 か grade:4 も一緒に書く。
2
グループ分けの変更
GROUPS の中のクラス一覧と試合の組み合わせを書き換える
場所 var GROUPS={ から始まるブロック
構造の説明
  a:{
    name:'aグループ',   // ← 表示名（変えてOK）
    cls:'grp-a',       // ← CSSクラス名（色。触らなくていい）
    teams:['4D','4E','5G','6B'],  // ← ★グループのクラス一覧
    matches:[                          // ← ★試合の組み合わせ一覧
      {id:'a1', h:'4D', a:'4E'},    // id=試合ID, h=ホーム, a=アウェー
      {id:'a2', h:'5G', a:'6B'},
      ...
    ]
  },
📝 変更ルール
・teams → そのグループに入るクラスを全部書く
・matches → 総当たりの全試合を書く
・id はグループ内でユニークならOK（a1, a2…など）
・4チームなら6試合、5チームなら10試合が総当たりの数
💡
5チーム（eグループ）の場合は10試合
4チーム×6試合、5チーム×10試合。試合の組み合わせはリーグ表をそのまま入力する。
3
トーナメント時間割の変更
SCHED の time を書き換えるだけ
場所 var SCHED=[ から始まるブロック
構造の説明
var SCHED=[
  {slot:1, time:'11:30~11:36', A:'Z1', B:'Z2'},
  {slot:2, time:'11:38~11:44', A:'Y1', B:'X1'},
  {slot:9, time:'12:34~12:41', A:'W4', B:null},  // B面なし → null
];
📝 変更ルール
・time の文字列を新しい時間に書き換えるだけ
・slot は試合番号（1から順番）
・A/B はA面・B面の試合ID（W1, X1, Z1…）
・B面がない試合（決勝など）は B:null
4
トーナメントの対戦組み合わせ
TM_SRC と TM_WREF を変更（構成が同じなら不要）
やや複雑 基本はサッカーと同じ構成なので変更不要なことが多い
TM_SRC：各試合の出場チームの出所
var TM_SRC={
  W1:[['b',1],['c',1]],  // W1 = bグループ1位 vs cグループ1位
  W2:[['a',1],null],    // W2 = aグループ1位 vs （前試合勝者）
  W4:[null,null],       // W4 = 前試合勝者 vs 前試合勝者
  Z1:[['a',4],['b',4]],  // Z1 = aグループ4位 vs bグループ4位
};
TM_WREF：前試合の勝者を参照する試合
var TM_WREF={
  W2:[null,'W1'],   // W2のB面チーム = W1の勝者
  W4:['W2','W3'],  // W4 = W2勝者 vs W3勝者（決勝）
  Z3:[null,'Z1'],  // Z3のB面チーム = Z1の勝者
};
✅
サッカー・バスケ・ドッヂボール・ドッヂビー・ハンドボールはすべて同じ構成
トーナメントの構成が変わらない限り、TM_SRC と TM_WREF は変更不要。
5
役割の種類・数の変更
renderShift() 関数内の pick の数と種類を変える
場所 function renderShift(){ の中

各試合で人を選ぶ関数が pick5first（5年優先）と pickAny（誰でも）の2種類あります。

役割の割り当て部分（試合ループ内）
// 審判を1人選ぶ（5年優先）
var ref = pick5first(refCount);
// ボールボーイを1人選ぶ（誰でもOK）
var bb  = pickAny(bbCount);
// 本部を1人選ぶ（誰でもOK）
var hb  = pickAny(hbCount);
📝 役割を追加したい場合
① カウント変数を追加: var 新役職Count={};
② 初期化に追加: ALL_STAFF.forEach(function(s){ 新役職Count[s.name]=0; });
③ pick を追加: var 新役職 = pickAny(新役職Count);
④ 表示部分にも追加（role-box のHTML）
関数意味使う場面
pick5first(cnt)5年生を優先して選ぶ主審・審判など重要な役割
pickAny(cnt)担当回数が少ない人から選ぶ副審・BB・本部・誘導など
6
Firebase保存先の変更
競技ごとにデータが混ざらないように名前を変える
重要 競技が違うのに同じ名前だとデータが上書きされる
変更する2箇所（必ず両方変える）
// データ読み込み（1箇所目）
db.ref('soccer').on('value', function(snap){ ... });

// データ保存（2箇所目）
db.ref('soccer').set({scores:scores, winners:winners}, ...);
📝 競技名の対応表（このシステムで使用中）
サッカー → 'soccer'
バスケットボール → 'basketball'
ドッヂボール → 'dodgeball'
ドッヂビー → 'dodgebee'
ハンドボール → 'handball'

新競技を追加するときは 'tagoni' など新しい名前を付ける。
7
デザイン・タイトルの変更
色・絵文字・タイトルを競技に合わせて変える
① ヘッダーの色（CSS内）
header{background:#185FA5; ...}   /* 青→サッカー */
/* バスケなら #993C1D（茶）、ドッヂビーなら #534AB7（紫）など */
② タブの色（CSS内）
.tab.on{color:#185FA5; border-bottom:2px solid #185FA5; ...}
③ タイトル文字（HTML内）
<header>
  ⚽ サッカー審判シフト  <!-- ここを変える -->
  <span>...</span>
</header>
📝 競技別おすすめ色
⚽ サッカー → #185FA5（青）
🏀 バスケ → #993C1D（茶）
🎯 ドッヂボール → #1D9E75（緑）
🥏 ドッヂビー → #534AB7（紫）
🤾 ハンドボール → #BA7517（金）

新しい競技を追加する完全手順
来年度の体実引き継ぎ用
1

既存のHTMLをコピーしてリネーム

例: soccer_shift.html をコピーして tagoni_shift.html などに名前を変える

2

テキストエディタで開く

メモ帳でもOK。Windowsなら右クリック→「プログラムから開く」→「メモ帳」

3

①〜⑦を順番に書き換える

上記ガイドの1〜7を確認しながら変更する。Ctrl+F（検索）で変数名を検索すると早い

4

ブラウザで動作確認

HTMLファイルをダブルクリックしてブラウザで開き、グループ戦の得点入力が動くか確認する（Firebaseは繋がらないが表示は確認できる）

5

GitHubにアップロード

soccer-siftリポジトリの「+」→「Upload files」からアップロード。Commit changesを押す。

6

URLをLINEで共有

https://nrkyagi1212-coder.github.io/soccer-sift/ファイル名.html でアクセスできる

⚠️
トラブルシューティング
症状原因対処法
ページが真っ白になる JavaScriptの構文エラー ブラウザのF12→コンソールでエラーを確認。カンマの付け忘れ・クォートの閉じ忘れが多い
審判が「不足」になる 出場クラスの体実が多くて使える人がいない その時間帯は人数不足。体実の人数を増やすか、出場クラスに関係なく使う人を追加する
スマホで同期されない FirebaseのルールがReadを許可していない Firebaseコンソール→Realtime Database→ルールで ".read": true, ".write": true に設定
勝者入力しても次の試合に反映されない TM_WREFのインデックスが間違い W2のB面は [null,'W1']（0番目がnull, 1番目がW1）になっているか確認
順位が正しく計算されない GROUPSのmatchesのidが重複している 全試合のIDがユニークか確認（a1, a2… b1, b2…のように重複しないようにする）
GitHubにアップしてもURLが404エラー GitHub Pagesのデプロイに時間がかかっている 2〜3分待ってからアクセスする。Settings→Pagesでデプロイ状況を確認
